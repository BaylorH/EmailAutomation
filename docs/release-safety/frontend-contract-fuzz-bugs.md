# Backend frontend-contract fuzz — bug inventory (2026-07-03)

Generated by a 12-agent adversarial fuzz of the frontend-facing `/api/*` route handlers in `app.py`, driving the REAL handlers via a local Flask test client + fake Firestore/Sheets/Graph (no live sends). Each `tests/test_fe_contract_fuzz_<route>.py` is RED where it pins a defect.

**332 mutations across 12 routes → 39 bugs (8 high / 23 medium / 8 low).** No recipient-guard breach on send paths (send boundaries never fired), EXCEPT `/api/trigger-scheduler` which has no auth at all.

## Systemic root cause (most mediums)
Handlers do no request-shape/type validation and wrap everything in `except Exception: return jsonify({'error': str(e)}), 500`. So non-dict JSON, malformed JSON, and wrong Content-Type become 500s that leak internal Python/werkzeug text; several error paths fail open (HTTP 200). Fix once with a shared `_require_json_object()` helper (safe parse → 400; validate field types; stop returning raw `str(e)`).
